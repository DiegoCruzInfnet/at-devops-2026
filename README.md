# AT DevOps 2026

## O papel do Git na Entrega Contínua

O Git é a base do devOps permitindo permite que times trabalhem simultaneamente no mesmo projeto, rastreando todas as mudanças e automatizem entregas de software.

## Branches

Branches são ramificações do código, onde cada nova funcionalidade ou correção é desenvolvida em uma branch separada, sem afetar o código principal. Quando pronta, é integrada via Pull Request.

## Tags e Releases

Tags marcam pontos importantes na história do projeto, como versões de software (v1.0, v2.0). Releases são distribuições oficiais do software associadas a essas tags.

## Git no ciclo de DevOps

- **Commit** → registra mudanças no código
- **Branch** → isola o desenvolvimento
- **Pull Request** → revisão de código antes de integrar
- **Tag** → marca versões estáveis
- **CI/CD** → workflows disparam automaticamente a cada push
